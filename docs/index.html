Compare-first control tower

Compare ChatGPT, Gemini, Perplexity, Qwen, and Grok from one local side panel.

Prompt Switchboard is a local-first browser extension and browser-side control tower that fans one prompt out to multiple supported AI chats, lines the answers up side by side, and keeps the trust boundary inside your browser.

The supported install path today is the packaged GitHub Release zip. Browser-store submission materials are being kept ready, but GitHub Releases remains the supported install surface today.

What changes in practice

The strongest evidence is not a vague promise about AI productivity. It is the before-and-after of the comparison workflow itself: less tab juggling, less copy-paste, and one place to review the run.

Manual comparison means duplicating prompts, waiting across tabs, and rebuilding context by hand. Prompt Switchboard reduces that overhead to one prompt fan-out, one compare board, and one saved local session.

The demo flow now shows the extension moving from ready state to compare fan-out to staged next-step workflow and completed side-by-side results.

The compare board now keeps next-step workflow, analyst guidance, model status, copy actions, and source-site shortcuts clear in one turn-level surface.

One browser-side workflow: open the supported sites, ask once from the side panel, and compare the answers without leaving your workspace.

Settings keep the surface polished for real use too: import/export, language, theme, and input behavior live in one panel.

First-success path

Install, compare once, then keep the turn alive.

What you need first: a Chromium-compatible browser with Developer Mode and at least one supported AI chat tab already open and signed in inside the same browser profile where you load Prompt Switchboard.

  1. Download the latest packaged build from the latest GitHub Release and unzip it locally.
  2. Open chrome://extensions, enable Developer Mode, and load the extracted folder as an unpacked extension.
  3. If the Prompt Switchboard icon is hidden, open the browser Extensions menu, pin it, and click the toolbar icon to open the side panel.
  4. Open your supported AI tabs, ask one prompt, and compare every answer from the side panel.
  5. Retry only the failed cards or export a readable compare artifact from that same turn.

Today the supported install path is the packaged GitHub Release zip. A lower-friction store install path is being prepared, but it is not live yet.

Official Google Chrome branded builds 137+ / 139+ no longer reliably auto-load unpacked extensions from command-line flags. Keep automated runtime proof on Chromium or Chrome for Testing. If you are validating in real Chrome, open chrome://extensions, enable Developer Mode, and use Load unpacked manually in the same signed-in profile.

After the compare

What you do next

  • Export a readable compare summary or Markdown artifact.
  • Retry only the failed models instead of replaying the whole turn.
  • Reuse prompt packs when you want a faster second run.

Still not the goal: a hosted relay, a provider-neutral SDK, or arbitrary site automation.
